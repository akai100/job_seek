## 主导 RISC-V 平台后端TRIM 功能的端到端交付

基于C语言在多核 RISC-V 平台实现后端 TRIM 功能全链路，覆盖前台TRIM、后台TRIM以及后台转前台TRIM三大路径。为提升 TRIM 性能，设计多级 FTL 表结构，包含 Cache TRIM Bitmap 表（缓存层）、DDR TRIM Bitmap 表（主存层）和 FTL L2P 表；

1. 前台 TRIM

+ 实现前台 TRIM 流程：TRIM 请求由硬件队列上送到 RISC-V CPU，对 FTL L2P 表进行批量无效标记；

+ 支持大范围请求切片处理，将子任务调度至其他 Core 并行执行；

+ 通过 GR（全局资源）计数器跟踪子切片完成状态，主线程在全部子任务结束后释放锁，确保 FTL 表修改的原子性与完整性。

2. 后台 TRIM

+ 实现对 Cache TRIM BITMAP表和 DDR TRIM BITMAP 表的修改


3. 后台转前台TRIM

+ 硬件模块定时扫描 Trim Bitmap 表，产生后台转前台Trim 发送到 硬件队列再上送到 RISC-V 上的CPU；

+ 完成对 Cache TRIM BITMAP 表和 DDR TRIM BITMAP 表的修改；

+ 批量完成对 FTL L2P 表的修改；

## 主导企业级 SSD 预读功能的端到端交付

+ 设计四态热点生命周期管理模型：构建了包含 Empty（空闲）、Occupied（占位监测）、Active & Running（全速预读）、Active & Stopped（挂起/收缩）的有限状态机（FSM），精准控制预读窗口的开启、展开、收缩与释放。

+ 实现动态热度评估算法：

  + 在 Occupied 态引入时间衰减机制，通过Timer API周期性检测IO活跃度，自动剔除“过冷”占位，防止无效资源占用；
 
  + 设定顺序读计数阈值，仅当连续IO命中且达到阈值时，才将热点晋升至 Active & Running 态并展开预读窗口，有效过滤随机读噪声
+ 构建写/Trim命令冲突处理机制：
  + 开发写命令与Trim命令对预读区间的实时拦截逻辑，一旦命中热点区间立即释放资源并重置状态，确保数据一致性并避免无效预读。
  + 集成FTL层状态反馈，当TMU（表管理单元）返回预取页非PPN（物理页号）状态时，自动将热点转入 Active & Stopped 态，停止新请求下发并逐步淘汰旧区间，实现资源的平滑回收。
+ 资源弹性调度优化：实现了预读窗口的动态伸缩策略，在负载趋冷时自动收缩窗口直至归零，最大化利用有限的片上SRAM/DRAM资源。
